# Додаток Б: Матриця тверджень і доказів

Цей додаток потрібен не для краси. Він знімає головний ризик технічної роботи для експертного захисту: сильні твердження мають бути швидко перевірними. Якщо твердження не веде до коду, перевірки віхи або команди відтворення, воно має бути або пом'якшене, або прибране.

## Як читати матрицю

* **Твердження** — твердження, яке книга дозволяє собі зробити.
* **Код** — контракт або реалізація, яка несе поведінку.
* **Тести** — сфокусовані тести, які перевіряють поведінкову межу.
* **Вимірювання / артефакт** — перевірка віхи, підсумок віхи або команда, яка дає доказ продуктивності чи релізу.
* **Межа** — межа, за яку твердження не виходить.

## Маршрут аудиту

```text
                 +--------------------------------+
                 | Сильне твердження у розділі   |
                 +--------------------------------+
                                  |
                                  v
                           +-------------+
                           | Є доказ?    |
                           +-------------+
                             |         |
                           ні|         |так
                             v         v
      +------------------------+     +--------------------------------+
      | Пом'якшити або         |     | Додати посилання на код,       |
      | прибрати твердження    |     | тест, перевірку віхи           |
      |                        |     | або її підсумок                |
      +------------------------+     +--------------------------------+
                                                  |
                                                  v
                                     +--------------------------------+
                                     | Додати команду перевірки,      |
                                     | якщо практично                 |
                                     +--------------------------------+
                                                  |
                                                  v
                                     +--------------------------------+
                                     | Назвати межу застосування      |
                                     +--------------------------------+
                                                  |
                                                  v
                                     +--------------------------------+
                                     | Записати незаявлене            |
                                     | твердження, якщо читач може    |
                                     | узагальнити надто широко       |
                                     +--------------------------------+
```

Цей маршрут важливий для стилю книги. Ми не прибираємо амбіцію; ми змушуємо кожне сильне речення пройти через доказ, команду або явно названу межу.
